> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a standalone text tool to `@fieldnotes/core` — click-to-place text boxes with element-level styling (fontSize, color, textAlign), naked appearance, and vertical auto-grow.

**Architecture:** New `TextElement` discriminated union member + `TextTool` class following existing NoteTool pattern. DOM-rendered (like notes). Reuses `NoteEditor` for inline editing with targeted viewport modifications for text-specific behavior (auto-height, empty cleanup).

**Tech Stack:** TypeScript, Vitest, jsdom

```typescript
import type { Tool, ToolContext, PointerState } from './types';
import { createText } from '../elements/element-factory';

export interface TextToolOptions {
  fontSize?: number;
  color?: string;
  textAlign?: 'left' | 'center' | 'right';
}

export class TextTool implements Tool {
  readonly name = 'text';
  private fontSize: number;
  private color: string;
  private textAlign: 'left' | 'center' | 'right';

  constructor(options: TextToolOptions = {}) {
    this.fontSize = options.fontSize ?? 16;
    this.color = options.color ?? '#1a1a1a';
    this.textAlign = options.textAlign ?? 'left';
  }

  setOptions(options: TextToolOptions): void {
    if (options.fontSize !== undefined) this.fontSize = options.fontSize;
    if (options.color !== undefined) this.color = options.color;
    if (options.textAlign !== undefined) this.textAlign = options.textAlign;
  }

  onActivate(ctx: ToolContext): void {
    ctx.setCursor?.('text');
  }

  onDeactivate(ctx: ToolContext): void {
    ctx.setCursor?.('default');
  }

  onPointerDown(_state: PointerState, _ctx: ToolContext): void {
    // Text is placed on pointer up
  }

  onPointerMove(_state: PointerState, _ctx: ToolContext): void {
    // No drag behavior for text placement
  }

  onPointerUp(state: PointerState, ctx: ToolContext): void {
    const world = ctx.camera.screenToWorld({ x: state.x, y: state.y });
    const textEl = createText({
      position: world,
      fontSize: this.fontSize,
      color: this.color,
      textAlign: this.textAlign,
    });
    ctx.store.add(textEl);
    ctx.requestRender();

    ctx.switchTool?.('select');
    ctx.editElement?.(textEl.id);
  }
}
```

- [ ] **Step 3: Add auto-height and empty cleanup via `onTextEditStop`**

The `NoteEditor.stopEditing()` saves text to the store, then calls the `onStop` callback (added in Task 5). The viewport uses this callback to:

1. Remove empty text elements
2. Measure `scrollHeight` and update `size.h` for non-empty text elements

**History note:** `NoteEditor.stopEditing()` calls `store.update(id, { text })` before the callback fires. For empty text cleanup, the `onTextEditStop` method pauses the history recorder before removing the element, so the remove is recorded as its own undo step. On undo, the user gets back an empty text element — this is acceptable behavior (they can then type or delete again).

Add a new method after `startEditingElement`:

```typescript
  private onTextEditStop(elementId: string): void {
    const element = this.store.getById(elementId);
    if (!element || element.type !== 'text') return;

    if (!element.text || element.text.trim() === '') {
      this.historyRecorder.begin();
      this.store.remove(elementId);
      this.historyRecorder.commit();
      return;
    }

    const node = this.domNodes.get(elementId);
    if (node && 'size' in element) {
      const measuredHeight = node.scrollHeight;
      if (measuredHeight !== element.size.h) {
        this.store.update(elementId, {
          size: { w: element.size.w, h: measuredHeight },
        });
      }
    }
  }
```

Wire it up in the constructor, after `this.noteEditor = new NoteEditor()` (line 55):

```typescript
this.noteEditor.setOnStop((id) => this.onTextEditStop(id));
```
